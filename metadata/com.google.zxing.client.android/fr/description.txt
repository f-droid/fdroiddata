Lisez les informations stockées dans les codes-barres ou créez des codes QR pour partager des adresses, des contacts ou du texte entre appareils.

Les recherches de livres, de produits et de sites Web sont effectuées sur un serveur Google (selon le pays d'origine choisi), bien qu'un bouton de recherche personnalisé puisse être créé dans les paramètres, par exemple ` duckduckgo.com/?q=%s` ou `wikibooks.org/w/index.php?search=%s`.

Prend en charge les types de codes-barres suivants :

* UPC-A et UPC-E
* EAN-8 et EAN-13
* Code 39 ; 93 ; 128
* ITF
* Codabar
* RSS-14 (toutes les variantes)
 * QR Code
 * Data Matrix
* Aztec (qualité 'beta')
* PDF 417 (qualité 'alpha')

Consultez le site Web pour le dépannage et des explications sur les autorisations. L’application vous permet de partager des contacts, des applications et des marque-pages dans un code QR. C’est pourquoi l’autorisation Contact est nécessaire. Voir le site «&nbsp;Visit Developer Website&nbsp;» ci-dessous ( https://github.com/zxing/zxing/wiki/Frequently-Asked-Questions ).

Si votre appareil ne scanne pas, essayez d’abord des solutions de rechange aux bogues d’appareils dans Paramètres. Activez-les toutes et essayez de les désactiver l’une après l’autre afin de déterminer celles qui sont nécessaires. Si rien n’y fait, essayez de vider le cache et les paramètres dans les paramètres d’Android.

Si celles-ci ne fonctionnent pas et que vous pensez qu'il s'agit d'un problème de l'application, envoyez un message à l'auteur avec les informations suivantes. Les messages sans ces informations ne pourront pas être traités, et seront donc ignorés.

- Type d’appareil
 –Une description sommaire de ce qui ne fonctionne pas : plantages ? Ne numérise pas ? Drôles d’apparence ?
 –Confirmation que vous avez d’abord essayé les étapes ci-dessus
– Une capture des journaux de l’appareil quand il démarre
